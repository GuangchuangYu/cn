大学教授力挺抄袭，强行洗地到裤衩都不要了，到底图什么？

August 19, 2017 in babble

连遮羞布都不要，难道这是个一被子的故事?

事情开始于我整理并曝光了西安电子科技大学一个讲师读在职博士期间抄袭了两篇文章，并顺利拿到学位晋升为副教授的故事。

这个故事很精彩

亮点1: 期间发现了一条抄袭链，GOSim被DOSim抄，而DOSim又被HPOSim抄，抄袭者都在文章中把抄来的算法实现当成自己做的东西来介绍。

亮点2: ppiPre抄袭GOSemSim的文章，我跟BMC Systems Biology反馈了，而编辑部竟然各种包庇，最后抄袭者两面三刀，和编辑部黑箱操作，在未告知我的情况下，出来一个洗地的Erratum，把抄袭说成不好意思，忘记引用了。

整个过程连载于以下四篇文章中：

  • [连载1]：学术期刊的学术不端，你见过吗？
  • [连载2]：Editor你的心不会痛吗？
  • [连载3]：辣眼睛，一篇抄袭引发的系列血案！
  • [连载4]：我像是一颗棋，进退任由你决定

如果想要分享这整个过程，可以分享以下这个汇总的文章：

  • 从业超过10年，未见过如此厚颜无耻之人

这个事情之所以精彩，还在于做为同行的华中科技大学薛宇教授，在科学网上发表文章，各种偷换概念，各种论据与论证不相关，强行为抄袭洗地。

  • 科学网文章：BMC Systems Biology没有学术不端

徐洲更同学当天写了文章回应：

  • 活久见，抄袭竟是重用他人代码没有致谢？

对于这种洗地洗到裤衩都不要的情况，我也是实在看不下去，只能为他们送出祝福：

  • 祝你们缠缠绵绵到天涯

然而事情还没完，X教授还是要以长辈的身份来跟大家普及一下《如何姿势正确的使用他人代码？》，手法如一，无非是偷换概念、论据与论证不相关。

GOSemSim做的是GO分析，ppiPre做的是相互作用预测，即使这两个代码相似度在99.9999999999%，那ppiPre也必然有自己独到的东西，不然如何能预测蛋白质的相互作用？所以抄袭是不存在的。

像上面这样的话也可以说，我只能说呵呵，X教授也列举了他自己以「正确的姿态」使用别人的代码，当然他自己也说了自己从不开源，只做webserver和database。做为webserver，是可以通过整合别人的工具来实现的，这不需要什么「原创性」，也不是什么scientific research，重点只在于降低门槛，提供有用的工具或流程。X教授当年博士毕业半年，就是以两篇NAR的webserver拿了中科大的副教授。对于拿来主义，大概也是深谙此道的！所以他是有资格来跟我们这些晚辈普及使用他人代码的正确打开方式的，大家肯定受益良多，X教授以亲身经历为我们打开了生信灌水的大门。

X教授所说的姿势正确，且不说里面有多少模糊概念甚至于偷换概念的东西，X教授没告诉我们的是什么姿势是学术所不能容忍的？我一直在文章中强调抄袭分两个层面，一是无视软件的开源协议（开源也是在版权的框架下的所谓反版权，开源协议本身也是版权协议），二是文章用了大篇幅在把别人做的东西当成自己做的来介绍（像ppiPre把GOSemSim实现的东西当成自己的原创实现来介绍。X教授把抄了别人的实现当成是各自实现，说是平行工作！偷换得一手好概念）。

X教授的观点无非是读书人的事情不能算偷，他的文章一直是用了别人的软件、用了别人的代码，而一直回避我强调的写文章说自己实现了别人的代码！你敢这么写，所谓的正确姿势（引用/致谢），全部都要失灵的，我不知道X教授是读博士期间，中科大没上过科学伦理课呢，还是去华中科技大学当教授，岗前培训没到位，似乎对「抄袭」的定义一无所知。X教授的文章也一直回避、无视这个最重要的点，我们永远无法唤醒一个装睡的人，我也不打算写文去掐了，反正X教授都是自说自话，持续在输出自己的正确姿势而已。

有学术修养的人，看完系列文章+X教授的回应，都应该有自己的评价。

用散点可视化一个矩阵

 Mar 3, 2018

当然是ggplot2画，才好和ggtree兼容，你首先问自己，抛开树，你能右边吗？本身你有个矩阵，row的顺序要跟着树来，你可以先不管，先随意，这是y轴。那x轴呢？你矩阵里的数字，打点你总该会的，而点连线，同样column的数字打出来的点，就连在一起，你先按这个思路画出来，再来问后续的。

这是我在「知识星球」中的回答，然而小伙伴表示还是不懂。那么我来详细讲解一下：

首先我们读入一个FASTA数据：

require(treeio)

fasta <- system.file("examples/FluA_H3_AA.fas", package="ggtree")
aa <- read.fasta(fasta)

数据用treeio包里的read.fasta去读这个氨基酸序列，它有个好处是你直接as.character(aa[[i]])就是个字母的向量，方便比较。

n = length(aa)
d = matrix(NA, ncol=n, nrow=n)
nm = labels(aa)
rownames(d) = colnames(d) = nm

for (i in seq_len(n)) {
    for (j in seq_len(i)) {
        d[i, j] = mean(as.character(aa[[i]]) != as.character(aa[[j]]))
        d[j, i] = d[i, j]
    }
}

这段代码生成了pairwise distance矩阵，你可以搞其它的统计量，都OK的。大家可以想一想，一般这种矩阵怎么可视化？用热图！热图用数字来填充颜色，而现在我们把数字当成x轴上的变量，然后打点，仅此而已，我们有没有办法画出来，当然可以。同样row的数值会在同一y轴上，如果是热图，同一column的数值会在同一x轴上，方便横向和纵向比较，而现在同一column的数值不在同一直线一了，因为我们打的点的位置使用了矩阵中的数值，同一column的点被拉扯到不同的位置上去了，所以为了能够比较或者是看清楚同一column上的点的走势，我们把同一column上的点用线条连接在一起。这就是小伙伴提问的图，就是这么画出来的，非常简单。我在「知识星球」里其实已经解答得很清楚了。

一条指令把统计+画图都给做了？似乎太美好！

 Mar 3, 2018

大家是否还记得我写的文章《画个小圈圈》？里面吐槽了ggpubr这个包的存在是制造混乱，徒增学习成本。我们知道ggplot2包里有个qplot函数，而在《ggplot2》的第一版中，hadley先用了一章介绍qplot，再介绍grammar of graphics，然后这个qplot就被吐槽，先入为主，给学习grammar of graphics，学习ggplot2语法制造了不必要的麻烦。然后如果你看《ggplot2》第二版，你就会发现qplot这一章被删掉了。我说ggpubr制造混乱，徒增学习成本并不是为了吐槽某公众号，而是实事求是。

我并不反对封装，我并没有强迫症说大家一定要用ggplot2图形语法，而不能封装函数，我想我在《画个小圈圈》已经说得很清楚了，封装画图的包，按我的观点是要么卖了数据处理的专业知识，要么卖了画图的专业知识。如果一个包做的封装太过于简单，像ggpubr这种，数据必须是用户整理好的data.frame，而画图只是简单的两三个图层的叠加，那么当然可以减少你敲键盘输入，然而影响了初级用户学习ggplot2，复杂的东西，我们当然想要用封装好的函数，等你自己啥都搞懂了，老板都要跟你掀桌子了。简单的东西也可以封装，减少输入嘛，只是特别不适用于ggplot2的图层，简单的设定你写个主题，我都举双手赞成，但两三个图层的简单拼凑我不支持，因为图层是积木，大家学习就是要对积木熟悉，然后才能搭出复杂的东西出来。特别简单的图层叠加正是你熟悉积木的时候，你不多敲点键盘，做为练习，又怎么能做出稍复杂点的东西呢？
