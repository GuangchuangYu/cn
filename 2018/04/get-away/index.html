滚吧，骚年

April 9, 2018 in life

4月3号发了以下一条文字推送：

遇到辣鸡导师，只有三个字，忍滚狠，我选择滚，考博期间，发现人品不好，我选择滚，《来香港读博其实是被逼的》，读博期间，本着就是退学也要滚的心态，把导师给炒了，《CS10: 八卦终结版》，忍下来也会消磨掉你所有追求/理想，比狠的话，如果没有压制对方的渠道，对于这种势力严重不均衡的情况似乎不太现实，所以我奉劝大家，滚！一两年的沉没时间，放长远了看，根本不算什么，要勇于放弃，滚的远远的

后台有不少人回复，索性发出来给大家看看，因为不是公开评论的，ID我就不写出来了。

Y叔，看了你今天的推送（仔细到连评论都看了不少），我想我就是你说的那种没有能力的人吧，知道是个坑可还是在坑里消磨了两年半的时光，并且还要继续下去……过得并不快乐，只是活着而已。想出去，却又怕得到的比现在更少。唉，我挺羡慕XZG的，他来我们组被拒绝了，但是他现在能做自己喜欢的事，并且有WJW老师的支持。他当初没有来我们组，真是冥冥中，天不让他受罪。

这个博士读的苦闷了，读的怀疑了人生，读的厌恶了生活。

读研究生都想滚了我

感慨前辈的坎坷啊，希望人生的每一步都会算数吧。

导师什么都不管，连实验室也没有，都想退学了

说的很有道理，我读博期间就遇到垃圾导师，期间想重新读，但是选择坚持下来，但现在看来，如果重新读，一定会很好，唉，现在把自己耽误了，后悔莫及

今年博四了，还在做。我老板以前跟着院士和千人，发了很多好文章，但是都是人家指导的好。很快成了博导，但是独立出来以后，没有任何的方向或者想法。

整个实验室已经10几个人毕业没有任何文章了

Y叔，你讲的一点也不错，我算倒霉，碰见的老板非常自满，最爱干的事就是在我们面前吹牛逼，其实他就一临床医生，发的文章还是外包公司搞的，有本事在我们院长面前吹呀，我们院长是院士。我学R之类的刚开始出去学习的人钱都是我自己掏的，他还BB我，MD，总有一天我要炒了他

推荐大家看这两部电影，根据真实事件改编，讲述的是无良变态老师虐待学生、性侵学生。《总有一天》里面的无良校长直到罪行被揭发的一刻都在对小男孩说：“我做的一切都是为你好。”

一个人作恶并不可怕，可怕的是一群人组成恶循环的链条，以我自己的事情来讲吧，我找了DRPC之后，DRPC两头了解了情况之后，找我说给我一个月的时间，保证前导师不骚扰我，照发奖学金，我必须要在这一个月时间内换导师，如果我找不到新的导师接收，我自己去退学。我答应了！DRPC开出来的这个条款，对我是不利的，要一个月换导师很难，多半是我退学了，而他们息事宁人，以最小的代价去保护前导师，如果把事情搞大，闹到学校里，显然都不好看。这是可能的，我去问过CEDARS，他们都跟我直接说，可以去Equal Opportunity告他，我当时问了说“难道Equal Opportunity不是保障残疾人的利益吗？”，他们回答我说，“不是的，就算是男人被性侵他们都管的”。处理我这件事情的DRPC还告诉了我前导师，叫他不要留下文字证据，于是后面的学生收集证据就更困难了，我同一级就两个学生（开门弟子，前面没有师兄师姐），另一个人后来也上DRPC了。所以说DRPC其实是在包庇纵容这些事情的！组团的恶势力就更恶了。

所以我觉得导师和学生的种种悲剧，学校是要负责任的，国内很多学校没有渠道去反馈，即使有也很可能是隐藏的，必须要有，而且在新生入学时进行介绍。像我在HKU，虽然DRPC纵容（可能因为同一department，抬头不见低头见），但还有别的渠道，CEDARS直接叫我去告前导师虐待。比如像《假期如何假装我们还在工作？》里暴光的邮件中暗示一天得干15小时：

I am not asking you to work at here from 10AM to 1AM, like what I did. But I do consider this factor when I write your bi-annual report to graduate school.

再者，我觉得学生也是有问题的，好比一个荷尔蒙过剩的小孩，恋爱分手的时候死去活来，为什么呢？因为他/她的世界就那么丁点大，「失去了你就失去了世界」嘛，而成年人，可能就「洗洗睡了，明天还上班呢」。做学术的人，很容易活在自己的小圈圈里面，大家其实应该交非学术圈的男女朋友，有助于把自己从学术中拉出来，接触点正常人的生活。人生的格局太小，容易钻牛角尖，如果生活是一个圆，要把它搞成个大圆，啥事情放长远了看，都不值一提，就像我开头说的，该沉没就沉没，该滚就滚。你改变不了导师，改变不了学校，你能改变的只有自己，要让自己看得长远，要跳出自己的小圈圈，你能够做的，就是重新选择，没有人能够虐一个放弃的人，我退学了，导师还能拿我怎么样！事情都是双向的，搞好自己这一头，大不了重头再来选一遍。

搞科研不容易，都是聪明人在烧脑袋的事情，最不缺的就是聪明的人，更不缺加班加点的，既然读研究生了，抗压能力也要有一些，如果是老板待人公正、做事理性，那么催工作进度，逼你干活什么的，你必须要抗住！如果像悲剧中那种家奴式的，公私不分，我劝你早点滚，从头再来。

用散点可视化一个矩阵

 Mar 3, 2018

当然是ggplot2画，才好和ggtree兼容，你首先问自己，抛开树，你能右边吗？本身你有个矩阵，row的顺序要跟着树来，你可以先不管，先随意，这是y轴。那x轴呢？你矩阵里的数字，打点你总该会的，而点连线，同样column的数字打出来的点，就连在一起，你先按这个思路画出来，再来问后续的。

这是我在「知识星球」中的回答，然而小伙伴表示还是不懂。那么我来详细讲解一下：

首先我们读入一个FASTA数据：

require(treeio)

fasta <- system.file("examples/FluA_H3_AA.fas", package="ggtree")
aa <- read.fasta(fasta)

数据用treeio包里的read.fasta去读这个氨基酸序列，它有个好处是你直接as.character(aa[[i]])就是个字母的向量，方便比较。

n = length(aa)
d = matrix(NA, ncol=n, nrow=n)
nm = labels(aa)
rownames(d) = colnames(d) = nm

for (i in seq_len(n)) {
    for (j in seq_len(i)) {
        d[i, j] = mean(as.character(aa[[i]]) != as.character(aa[[j]]))
        d[j, i] = d[i, j]
    }
}

这段代码生成了pairwise distance矩阵，你可以搞其它的统计量，都OK的。大家可以想一想，一般这种矩阵怎么可视化？用热图！热图用数字来填充颜色，而现在我们把数字当成x轴上的变量，然后打点，仅此而已，我们有没有办法画出来，当然可以。同样row的数值会在同一y轴上，如果是热图，同一column的数值会在同一x轴上，方便横向和纵向比较，而现在同一column的数值不在同一直线一了，因为我们打的点的位置使用了矩阵中的数值，同一column的点被拉扯到不同的位置上去了，所以为了能够比较或者是看清楚同一column上的点的走势，我们把同一column上的点用线条连接在一起。这就是小伙伴提问的图，就是这么画出来的，非常简单。我在「知识星球」里其实已经解答得很清楚了。

一条指令把统计+画图都给做了？似乎太美好！

 Mar 3, 2018

大家是否还记得我写的文章《画个小圈圈》？里面吐槽了ggpubr这个包的存在是制造混乱，徒增学习成本。我们知道ggplot2包里有个qplot函数，而在《ggplot2》的第一版中，hadley先用了一章介绍qplot，再介绍grammar of graphics，然后这个qplot就被吐槽，先入为主，给学习grammar of graphics，学习ggplot2语法制造了不必要的麻烦。然后如果你看《ggplot2》第二版，你就会发现qplot这一章被删掉了。我说ggpubr制造混乱，徒增学习成本并不是为了吐槽某公众号，而是实事求是。

我并不反对封装，我并没有强迫症说大家一定要用ggplot2图形语法，而不能封装函数，我想我在《画个小圈圈》已经说得很清楚了，封装画图的包，按我的观点是要么卖了数据处理的专业知识，要么卖了画图的专业知识。如果一个包做的封装太过于简单，像ggpubr这种，数据必须是用户整理好的data.frame，而画图只是简单的两三个图层的叠加，那么当然可以减少你敲键盘输入，然而影响了初级用户学习ggplot2，复杂的东西，我们当然想要用封装好的函数，等你自己啥都搞懂了，老板都要跟你掀桌子了。简单的东西也可以封装，减少输入嘛，只是特别不适用于ggplot2的图层，简单的设定你写个主题，我都举双手赞成，但两三个图层的简单拼凑我不支持，因为图层是积木，大家学习就是要对积木熟悉，然后才能搭出复杂的东西出来。特别简单的图层叠加正是你熟悉积木的时候，你不多敲点键盘，做为练习，又怎么能做出稍复杂点的东西呢？

扪心自问，meme几何？

 Mar 3, 2018

很多人都在学ggplot2，也有很多「高手」，比如你搜索「精通ggplot2」，你甚至会看到各种「从入门到精通」的课程，然而都是各种照猫画老虎的入门菜谱而已，当然广大群众去上课也可能只是为了求菜谱好在需要的时候，可以copy-paste而已。

ggplot2学习容易，因为无非是图层叠加，而图层的参数一致性非常好，网上也有很全的文档。然而要深入很难，比如你想干点什么是ggplot2自身没有支持的，你需要自己去hack，需要用到更底层的东西的时候，一般情况下，会无从入手，尝试之后才知道什么叫绝望！因为ggplot2套路实在是太深。

正如我在《听说你还不会画热图》吐槽的，有些所谓的「高手」实际上连ggplot2底层是什么都不知道！然后可以把ggplot2吹上天。很多人根本不知道当你打下ggplot这个命令的时候，到底发生了什么？要理解ggplot2并不容易，正如前面所说「套路太深」，没有金钢钻，读不了它的代码，有没有一个简单点的包，能够读懂代码，并且有助于理解ggplot2呢？我想我的meme包就非常好，虽然仿的有点肤浅，然而胜在简单，看过下面这两篇文章的，都知道这个包在做什么。
