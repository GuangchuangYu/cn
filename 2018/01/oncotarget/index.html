我被oncotarget撞了腰

January 14, 2018 in Biology

最近似乎就oncotarget是否要被SCI踢出来这事情，真真假假的报道挺多。以我的观点来看，它被踢是正常的；它不被踢也迟早要被踢！就像当年它的影响因子还在7点多的时候，我就预言oncotarget药丸一样，结果就直接掉到5点多，然后关于神刊的报道就开始多了起来。

话说我在某高校工作的时候，有一个同事，他跳槽了！跳槽前各种暗渡陈仓，压着文章不发表，蓄谋已久的事情，然后就在跳槽的当年，可能为了掩饰，当年还继续招生，结果当然坑惨了那一届的学生，当然前面两三届也惨，因为数据压着不发表。

虽然离职前和旧单位签了协议，数据是在旧单位做的，发表文章旧单位必须是通讯作者，仅此而已，反正都是他通讯，挂两单位而已嘛，这在学术界再正常不过。此人去了新单位之后，本来该发文章了，原来的学生干了几年，也该给人一个authorship了，然而他把新单位的领导放在了第一作者，自己当了通讯，原来的学生嘛，见鬼去了，其他作者多半是他的新同事/学生。当然通讯也没挂原单位，对于没有社会契约精神的人来说，协议就是一张废纸。

这个事情就和我有关了，因为我是那干过活的老同事啊，文章当然也没有挂我的名字了。我在老单位，对文章挂名还是挺在意的，我在意的是别人是否对我的工作有一个认可，而不是在意文章本身，我工作几年，发的文章多半是自己做的，自己第一作者。而别人的文章，我从未跟别人讲过把我名字放前一点，因为第一我觉得学生比我更需要，第二我干活是有工资拿的。但我有参与了，文章中路人甲的位置还是要给我，所以即便是之前有学生发中文文章，她认为我看不上，没写我名字，我还是要让她写上去，因为我有帮她做了分析。其实我是真看不上，但我希望告诉别人，要对我的工作有一定的认可。

那某人之所以更过份，是因为他离开了旧单位，而我也来了港大，这时候他还经常找我分析数据，我帮他做了，文章竟然没有挂我名字，这就有点意思了，以前我可以说我做为单位的bioinformatician拿着工资在办事，现在大家不在一个单位，没有任何瓜葛，我还义务给你做分析了不成！于是我写信一封过去，说我发现了尼玛用了我的分析结果，没有挂我名字，连个致谢都没有，以后不许再用我的分析结果，否则我要联系杂志！这只是个warning，并且我只是私下发给他自己，咱也是先礼后兵，日后好相见嘛。

然后～～更有意思的事情来了，他又用了我的分析结果，然后在致谢里感谢我做了统计和生信分析，尼玛这就是在我头上拉屎拉尿了，不可忍！但我还是没有直接联系杂志，而是再给他写信，说你这样太过份了，我给你留足了面子，没有把信抄送给你的老领导和新领导，也没联系杂志，私下给你警告了，你竟然完全无视。当然这一封我抄送给了他新老领导。然后他给我怼回来，说看了我之前的信，感觉我对于文章挂名看得很重，他已经按我说的给我致谢了（这智商我服，肯定是故意曲解），然后又说以前和我合作同一的文章，是他卖了人情给我，我现在就应该还他人情。这逻辑，我又服！讲真那篇同一，挂不挂同一在他不在我，他觉得给我人情，没问题。但问题是那篇蛋白质组学的文章，质谱是技术员做的，分析是我做的，学生做了分子实验验证一下，把我放在学生后面当同一，我并不是平白无故拿的同一，我并不觉得我欠什么人情，而且好多都这种套路的组学文章，我做了多少篇，从来都是挂在路人甲的位置，我从来没说过一句话，我给出的人情少么？扯这些就没意思，还是我原来说的一点，现而今大家在两个单位，没有任何瓜葛的情况下，发生的关系，你文章没挂名，也没给钱吧，难道你要来肉偿吗？丑拒！

于是发生了我和oncotarget的联系，他两篇文章都发oncotarget，我于是联系了editor，说明了情况，而且提供给editor数据和图表，没人理我，于是我联系了好几个editor，最后有一个回复我了，说abuse of authorship这种事情，我们是不管的，你们单位自己去管。我就不再联系了，因为我知道不值得，跟这种破烂杂志纠缠简直就是浪费生命。

话说我在港大之后，之前有个学生发文章，再上线，我看到有我做的图，我也没去警告她，只是微博上吐槽一下，说人走茶凉之类的，她看到了，立刻联系我跟我道歉，说疏忽了，要联系杂志看还能不能加上去，最后还真加了个路人甲的位置，这样其实大家的关系还更进一步，好感度是增加的。上面故事中的这个人，就是个SB，当时警告他，他没回信，过后还继续搞，这不是逼着我要出来干一架么！

科研干久了，奇葩总是会遇到的，以前在读硕士的时候，帮同学做过一个分析，她后面联系我说她同学要做类似的，介绍给我，反正我代码是有的，随便喊个价就好了。我当时跟她同学说，我帮她做了，到时候按贡献署名就好了，当交个朋友，她第二天跟我说跟她老板说了，后面一定把我大名放在前面，其实前不前面都是路人甲，挂名的文章都不是硬通货。那时候已经是过了几年，之前的代码早不见了，所以我是重新又做了一遍帮她做的，然后就完事了，过了许久，因为是过节群发信息问候吧，我收到后就顺道问她进展怎么样啊，她告诉我说她们老板觉得这是篇大文章，不能挂名，我问她什么样的大文章，她回答我说5分。我表示呵呵，尼玛5分好意思说大文章，然后过一阵子，我收到了一份editor给所有作者发的确认信，有一篇我没有参与的小文章，我挂在倒数第二，单位还是她们的单位，不是我的单位，于是我给editor写信说，我没参与，请把我的名字删掉。

用散点可视化一个矩阵

 Mar 3, 2018

当然是ggplot2画，才好和ggtree兼容，你首先问自己，抛开树，你能右边吗？本身你有个矩阵，row的顺序要跟着树来，你可以先不管，先随意，这是y轴。那x轴呢？你矩阵里的数字，打点你总该会的，而点连线，同样column的数字打出来的点，就连在一起，你先按这个思路画出来，再来问后续的。

这是我在「知识星球」中的回答，然而小伙伴表示还是不懂。那么我来详细讲解一下：

首先我们读入一个FASTA数据：

require(treeio)

fasta <- system.file("examples/FluA_H3_AA.fas", package="ggtree")
aa <- read.fasta(fasta)

数据用treeio包里的read.fasta去读这个氨基酸序列，它有个好处是你直接as.character(aa[[i]])就是个字母的向量，方便比较。

n = length(aa)
d = matrix(NA, ncol=n, nrow=n)
nm = labels(aa)
rownames(d) = colnames(d) = nm

for (i in seq_len(n)) {
    for (j in seq_len(i)) {
        d[i, j] = mean(as.character(aa[[i]]) != as.character(aa[[j]]))
        d[j, i] = d[i, j]
    }
}

这段代码生成了pairwise distance矩阵，你可以搞其它的统计量，都OK的。大家可以想一想，一般这种矩阵怎么可视化？用热图！热图用数字来填充颜色，而现在我们把数字当成x轴上的变量，然后打点，仅此而已，我们有没有办法画出来，当然可以。同样row的数值会在同一y轴上，如果是热图，同一column的数值会在同一x轴上，方便横向和纵向比较，而现在同一column的数值不在同一直线一了，因为我们打的点的位置使用了矩阵中的数值，同一column的点被拉扯到不同的位置上去了，所以为了能够比较或者是看清楚同一column上的点的走势，我们把同一column上的点用线条连接在一起。这就是小伙伴提问的图，就是这么画出来的，非常简单。我在「知识星球」里其实已经解答得很清楚了。

一条指令把统计+画图都给做了？似乎太美好！

 Mar 3, 2018

大家是否还记得我写的文章《画个小圈圈》？里面吐槽了ggpubr这个包的存在是制造混乱，徒增学习成本。我们知道ggplot2包里有个qplot函数，而在《ggplot2》的第一版中，hadley先用了一章介绍qplot，再介绍grammar of graphics，然后这个qplot就被吐槽，先入为主，给学习grammar of graphics，学习ggplot2语法制造了不必要的麻烦。然后如果你看《ggplot2》第二版，你就会发现qplot这一章被删掉了。我说ggpubr制造混乱，徒增学习成本并不是为了吐槽某公众号，而是实事求是。

我并不反对封装，我并没有强迫症说大家一定要用ggplot2图形语法，而不能封装函数，我想我在《画个小圈圈》已经说得很清楚了，封装画图的包，按我的观点是要么卖了数据处理的专业知识，要么卖了画图的专业知识。如果一个包做的封装太过于简单，像ggpubr这种，数据必须是用户整理好的data.frame，而画图只是简单的两三个图层的叠加，那么当然可以减少你敲键盘输入，然而影响了初级用户学习ggplot2，复杂的东西，我们当然想要用封装好的函数，等你自己啥都搞懂了，老板都要跟你掀桌子了。简单的东西也可以封装，减少输入嘛，只是特别不适用于ggplot2的图层，简单的设定你写个主题，我都举双手赞成，但两三个图层的简单拼凑我不支持，因为图层是积木，大家学习就是要对积木熟悉，然后才能搭出复杂的东西出来。特别简单的图层叠加正是你熟悉积木的时候，你不多敲点键盘，做为练习，又怎么能做出稍复杂点的东西呢？
